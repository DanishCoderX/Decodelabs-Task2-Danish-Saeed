I'm completing DecodeLabs Frontend Development Batch 2026. Project 2 focuses on responsive design — making layouts that work seamlessly on every device, from a 320px phone to a widescreen monitor.

The mobile-first strategy means I write base CSS for small screens, then use @media (min-width: …) to progressively enhance for larger viewports.

  • 02 Fluid units: %, rem, vw, clamp()

Core Skills

The responsive toolkit.

📐

Mobile-First CSS

Base styles target the smallest screen. Complexity is layered on with min-width queries — never the other way around.

📏

Fluid Typography

clamp(1rem, 2.5vw, 2rem) replaces stepped font-size breakpoints with a smooth, continuous scale.

☰

Popover Navigation

The HTML Popover API powers a full-screen mobile menu with a backdrop — zero JavaScript required.

⊞

CSS Grid

repeat(auto-fit, minmax(250px, 1fr)) creates a self-adjusting grid with no media queries needed.

↔

Flexbox

One-dimensional alignment for nav bars, button groups, and card internals. Direction flips at breakpoints.

👆

Touch Targets

Every interactive element meets the 44×44px minimum. Zoom is never disabled. WCAG compliant throughout.

Breakpoints

Bridges, not fences.

Mobile
0px → 479px

Single column. Stacked layout. Hamburger nav. Full-width tap targets.

Tablet
480px → 767px

Two-column grid. Layout enhancements. Side-by-side content blocks.

Tablet+
768px → 1023px

Desktop nav appears. Three-column grids. Full sidebar layouts.

Desktop
1024px+

Full grid. High-res assets. Maximum content width capped at 1200px.
